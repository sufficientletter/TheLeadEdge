Scoring Methodology
Multi-signal weighted scoring with temporal decay

Base Points

②
Temporal Decay
Signals lose value over time (4 decay types)
③
Freshness Premium
Signals within 24 hours get 1.3x-1.5x boost
④
Signal Stacking
Signal combos trigger 1.4x-2.5x multipliers
⑤
Normalization
Raw points scaled to 0-100 range
⑥
Tier Assignment
S / A / B / C / D classification
Temporal Decay Functions
Exponential Decay
Fast decline for time-sensitive signals (expired listings, price reductions). Half-life: 14-21 days.
Linear Decay
Steady decline for persistent conditions (foreclosure, absentee owner). Half-life: 90-180 days.
Step Decay
Full value for a period, then drops sharply (severe price reduction, high DOM). Half-life: 30-60 days.
Escalating
Increases in urgency over time (pre-foreclosure approaching auction date). Time-pressure amplifier.

Technical Notes
Known issues and data quality observations
Non-Critical CSV Field Mapping Issue

The sample CSV's DaysOnMarket column has an empty value for row 1, but a duplicate CumulativeDaysOnMarket value (45,45) is causing a column alignment shift in some rows. This results in city/zip_code/county being stored in incorrect columns for some records.

Impact: Display data (city names, ZIP codes) may appear incorrect in the database for the test dataset. Signal detection and scoring are unaffected because they operate on the raw record before storage.

Fix: This is a test data issue, not a code bug. Real Matrix CSV exports from SWFLAMLS will have consistent column alignment.

Non-Critical Signals Not Firing (22401007, 22401008)

The foreclosure flag (MLS# 22401007) and short sale flag (MLS# 22401008) are set in the raw CSV (ForeclosedREOYN=Yes, PotentialShortSaleYN=Yes) but are not being detected as signals. This is likely related to the field mapping issue above — the boolean columns may be shifted.

With real MLS data these will fire correctly.

  3. High Priority Scheduled Automation (Phase 2)
    Currently the pipeline runs manually. To maximize value:
    • APScheduler integration — Auto-run import + score + briefing daily at 6:00 AM
    • File watcher — Detect new CSVs dropped into data/mls_imports/ and auto-process
    • Re-scoring schedule — Re-score all active leads every 6 hours to capture decay changes
  4. High Priority Public Records Data (Phase 2)
    Five additional signal types are configured but have no data source yet:
    • Pre-Foreclosure (20 pts) — Lis Pendens filings from County Clerk
    • Tax Delinquent (13 pts) — County Tax Collector records
    • Code Violation (12 pts) — City/County code enforcement
    • Probate (18 pts) — County Probate Court records
    • Divorce (16 pts) — County domestic relations filings
    These combined with MLS signals trigger the most powerful stacking rules (2.0x-2.5x multipliers).
  5. Medium Priority Historical Data for Trend Detection
    With historical imports over time: price trajectory tracking, DOM velocity vs. neighborhood median, re-listing detection (same property under new MLS number), and seasonal pattern calibration for SWFLA.
  6. Medium Priority CRM Integration (Phase 4 — Follow Up Boss)
    Auto-create contacts for A-tier leads, trigger action plans/drip sequences, track conversion outcomes to calibrate scoring weights, and close the feedback loop.
  7. Ongoing Scoring Calibration
    Track conversion outcomes, A/B test tier boundaries, tune decay rates, and develop market-specific weights. All weights, thresholds, and decay rates live in config/scoring_weights.yaml and can be tuned without changing code.
